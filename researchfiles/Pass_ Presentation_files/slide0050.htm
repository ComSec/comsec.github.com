n“I/O Completion Ports” model vs One thread per connection
nDecision Made:  “I/O Completion Ports” model
¡Pros
nI/O Completion Ports design minimizes the problem of threads blocking at synchronous network operations, which improves performance scalability and resource usage
¡Cons
nMore complex design
nHarder to debug
n
n
Once an asynchronous network operation is issued, the thread is returned to the thread pool immediately. As soon as the network operation completes, the thread pool will automatically retrieve an available thread to continue execution.

Analysis of Known-Key Attack
  • Why the attack works
    • DSA and ECC-DSA satisfy homomorphic condition (II)



  • Why it doesn’t work in LFSR-based schemes
    • 2nd–order and 3rd-order characteristic sequences satisfy (III) and (IV), resp.






    • LUC, GH, and XTR, do not satisfy (II) and are thus all resistant to the known-key attack!

Pre-Authentication Cost
  • Zhou & Haas scheme
    • Uses threshold scheme,         (t+1) server nodes = CA


    • Comments:
      • Large overall computation and communication cost
      • Special role of server nodes interferes with design goals
      • Heavy workload & memory requirement for server nodes
      • Requires large number of servers in communication range
25
 Pre-Authentication Cost
  • Hubaux et. al scheme
    • PGP-like
    • Every node stores out- and    in-bound list of certificates
    • Comments:
      • Cheap set-up phase, cost controlled by node itself
      • A needs to verify more than 1 certificate
      • Performance depends on path length
        • network structure, # of nodes, # of stored certificates
      • Requires large number of nodes
      • Requires large memory

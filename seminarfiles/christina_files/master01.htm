This diagram shows the components in a spread spectrum communication system. 
At first, the source signal is modulated using BPSK for example, then it is multiplied by the sequence generator which generates the unique PN code.  The spreaded signal is then transmitted through a noisy channel.  At the receiver side, the received signal is again multiplied by the same PN code.  The despread signal is then passed through the BPSK modulator and matched filter.  The decoded data bit is then sent to detector to for bit 1 or 0 decision making.
This figure shows the effect of spreading and despreading that can reject interference/jamming signal that are introduced to the signal in the channel.  At the transmitter, here the source signal has low power density due to the spreading, and the noise in the channel has high power peak.  When the noisy signal is despreaded at the receiver, the effect of PN code multiplication “spreads” the noise, while the original transmitted signal is obtained.  When this is passed through data-detection, the jamming effects are already been reduced, which leads to more reliable signal detection. 
System capacity is defined by how many users the system can support, which is equal to the total number of unique signature sequences that can be used to identify all users with minimal cross correlations in the system.
A CDMA receiver separates determines which user the signal comes from by means of a correlator.  This figure shows the structure of a correlator receiver.  At each branch, the received signal is despreaded with a particular binary PN sequence and collect the energy of the despread signal.  If the PN sequence that used to despread is not the same as the spreading sequence, the signal will not be despread successfully, and as a result, only contribute to the noise.   Therefore, only the despread signal that is despread using the correct PN code will have maximum signal energy.  The receiver is then able to tell where the signal is from and detect it.  The output of the correlator is sent to a narrow-bandwidth filter.
Note that the form of match filter here is the equivalent to computing the cross correlation between the received signal and PN code.  When the despread code matches the spreading code, it is the same as computing the autocorrelation, which should have a peak when the 2 sequences have the same time-shifts.
Given N subcarriers to spread the baseband signal, we need chip duration that is N times longer than single-carrier CDMA to spread it across the same bandwidth.  Require lower speed of signal processing, lower power.  With low rate of chip, each subcarrier experience non-frequency selective fading.

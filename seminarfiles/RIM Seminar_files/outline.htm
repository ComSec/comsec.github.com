Motivations
  • Cryptosystems that use characteristic sequences have small modulus and high computations speed compared to some existing systems with same degree of security.
    • This is desirable for constrained devices!

Algorithms for Computing Sequence Terms
  • Three algorithms for computing sequence term in a 3rd order characteristic sequence:
    • Algorithm 1: Dual-State Fast Evaluation Algorithm
      • To compute the s±(k-1) state
    • Algorithm 2: Computation of a Previous Sequence Term
      • To compute s±(k-1) using (sk, sk+1) and it dual
    • Algorithm 3: Computation of Mixed term
      • To compute s±u(k+v)

GH-DSA: Signing
  • System Parameters:
    • f(x) = x3  ax2 + bx  1, an irreducible polynomial over GF(p),
    • where p is a prime number.  Period is Q is a factor of p2 + p + 1.

  • Signing process:
    • Compute public key pair (Algorithm 1).
    • Randomly choose k, with 0 < k < Q and gcd(k,Q) = 1.  Compute sk (Algorithm 1).  Set r = sk if gcd(sk,Q) = 1.
    • Compute h = h(m), where m is the message to be signed and h(·) is a hash function.
    • Compute t = k-1(h - rx) mod Q.

  • (r, t) is a digital signature of the message m.  Alice sends (m,r,t), (sx, sx+1), (sk, sk+1) and their duals to Bob.
8
GH-DSA: Verifying
  • Signing Equation h º xr + kt mod Q
  • Two cases:
    • If gcd(t, Q) = 1 Þ Case 1, else Þ Case 2

  • Case 1:
    • Compute s±(k-1) using (sk, sk+1) and its dual (Algorithm 2).
    • Compute u = -r-1t mod Q, v = -ht-1 mod Q.
    • Compute mixed terms s±u(k+v) (Algorithm 3).
    • Accept signature if s±u(k+v) = s±x.

Platforms
  • GH-PKC is implemented on a personal computer (PC) and a RIM Blackberry device:
    • PC:
      • Pentium 4 PC with 2.4 GHz processor and 512 MB of RAM.  C++ and Java implementations are ran under Windows XP.
    • Blackberry:
      • RIM Blackberry 6210 wireless handheld that has a 32-bit microprocessor with 16MB flash memory plus 2MB SRAM.

Analysis: Key Agreement
  • GH-DH is about 60% and 40% faster than DH on PC and Blackberry respectively.
  • Key length of GH-DH is only two-third of that in DH!

Analysis: Signing Procedure
  • GH-DSA is slightly faster than the DSS on both PC and Blackberry

Analysis: Verifying Procedure
  • GH-DSA is slightly faster than the DSS on Blackberry and is about 20% faster on PC.

Remarks and Future Research
  • The GH-PKC is implemented in C++ and Java and Blackberry JDE.  Analysis is done with Java implementation on PC and on the Blackberry simulator.
  • Performance improvement of GH-DH key agreement protocol over the traditional DH-key agreement protocol can be seen clearly.
  • Although improvement of GH-DSA cannot be seen easily, the process time is comparable to that of DSS.
  • Improvements can be more easily seen in hardware implementation as the modulus length in GH-PKC is only one-third of that in DH and DSS.  Future research!

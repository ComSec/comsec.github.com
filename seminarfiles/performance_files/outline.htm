4
Approaches and their problems
  • Symmetric:
    • Shared keys required
  • Hybrid:
    • Exchange of some authentic data required
    • e.g., Hash Chains: cannot establish session key
  • Asymmetric:
    • No TTP available
      • certificate issuing, revoking, renewing
    • Some heavy computations required

      • Sequence Term Calculation in LUC, GH, XTR (STC)
  • Bandwidth
  • Memory requirement
9
PKI Models
  • Distributed CA Model
    • Tasks of CA distributed to t network nodes
    • Power of CA distributed => no single point of attack
    • Implemented by (t,n)-threshold scheme
  • Self-Organization Model
    • Nodes issue and distribute certificates
    • Generates trust in PGP manner

Conclusions I
  • General:
    • Pre-Authentication and Authentication Performance can be analyzed separately
    • Authentication-Cost fixed for particular DSS
    • Seeking for efficient ad hoc network protocol
16
Conclusions II
  • Pre-Authentication
    • The 2 considered protocols:
      • do not take the constraintness of the devices into account!
      • do require large number of nodes
    • Threshold schemes require large communication & computation overhead
  • Authentication
    • No scheme ideal w.r.t all properties
    • Trade-off between computation costs, memory requirements and bandwidth

Finite Field Cryptography
  • Finite field GF(q)* with q = pr, p prime
  • Normally use q = p prime
  • Choose β in GF(q)* with order Q, i.e.       1, β, β2, …, βQ-1 are all different

Finite Field Bandwidth
  • Minimum requirements for security:
    • Q > 2160  (general group attacks)
    • q > 21024  (index calculus attacks)
  • Need 1024 bits to send 160 bits of data
  • Can we do better?
6
Reduced Representations
  • Idea: Represent finite field elements by minimal polynomials (sequences)
  • E.g. q = p3, β has minimal polynomial
  • x3 – ax2  + bx – 1
  • over GF(p) (Gong-Harn cryptosystem)
  • Transmit (a,b) only 2/3 as many bits
  • Computation in GF(p)
7
More Reduced Representations
  • Other examples:
    • q = p2, need 1/2 as many bits (LUC)
    • q = p6, need 1/3 as many bits (XTR)
    • q = p10, need 2/5 as many bits (TBA)
  • q = p10 referred to as fifth-order case
  • subject of current research
8
Fifth-Order Case
  • Computation in GF(p) with 102-bit p
  • Fast exponentiation algorithm found
  • Signature scheme based on Trace-DLP
  • To be presented at ISIT 2004

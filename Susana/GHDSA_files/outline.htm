GH Public-key Cryptosystem (GH-PKC) Overview
  • GH-PKC is based on the 3-rd order LFSR sequence with particular phase.
  • GH Diffie-Hellman (GH-DH) key agreement protocol published in proceeding of ChinaCrypto1998.
  • GH-DH / GH-DSA was published in  proceeding of the 8th Annual Workshop on Selected Areas in Cryptography, Aug 2001
  • Security is based on the difficulty in solving DL problem in GF(q3) for q = p or q = p2

GH-DSA
  • ElGamal-like signature algorithm


  • System Public Parameters:
    • Irreducible polynomial f(x) = x3  ax2 + bx  1 over GF(p), where p is a prime number, with period Q
  • Alice:
    • Private Key: Choose x, with 0 < x < Q and gcd(x, Q) = 1
    • Public Key: The s±x terms generate by f(x)
19
GH-DSA: Signing Process
  • Signing Process:
    • Randomly choose k, with 0 < k < Q and gcd(k,Q) = 1.  Use DSEA algorithm to compute (sk-1, sk, sk+1) and its dual such that gcd(sk, Q) = 1.  r = sk
    • Compute h=h(m), where h() is a hash function
    • Solve for t in the signing equation: h º xr + kt mod Q Þ t º k-1(h - rx) mod Q
  • (r, t) is a digital signature of the message m.  Alice sends Bob (m, r, t) together with (sk, sk+1) and its dual.
20
GH-DSA: Verification Process
  • Signing Equation h º xr + kt mod Q
  • But, Bob can verify Sequence Terms
  • Verification Process:
    • Compute s±(k-1) using (sk, sk+1) and its dual
    • If gcd(t, Q) = 1 Þ Case 1, else Þ Case 2
  • Case 1:
    • Compute u = -r-1t mod Q, v = -ht-1 mod Q
    • Compute mixed terms s±u(k+v)
    • Verify sequence terms

GH-DSA: Verification Process
  • Case 2:
    • Compute u = -r mod Q, v = -hr-1 mod Q
    • Compute mixed terms s±u(x+v)
    • Compute s±kt
    • Verify sequence terms

Software Design
  • Signing Process:
    • Input private key x.  Compute (sx, s-x).
    • Input signing key k.  Compute (sk-1, sk, sk+1) and it dual.  If gcd(sk, Q) ¹ 1, prompt user to choose another k.
    • If delta = 0, prompt user to pick another k.
    • Input message m to be signed.  Assume h = m.
    • r = sk, Compute t.
    • If t = 0, prompt user to choose another k.
    • Output (sx, s-x), (m, r, t), (sk, sk+1) and its dual

Software Design
  • Verification Process:
    • Input (m, r, t), (sx, s-x), (sk, sk+1) and its dual.
    • Compute s±(k-1).
    • If gcd(t, Q) = 1, then
      • u = -r-1t mod Q, v = -ht-1 mod Q
      • Compute s±u(k+v)
      • Verify if s±u(k+v) = s±x
    • If gcd(t, Q) ¹ 1, then
      • u = -r mod Q, v = -hr-1 mod Q
      • Compute s±u(x+v)
      • Compute s ±kt
      • Verify if s±u(x+v) = s±kt
    • Accept signature if sequence terms match

Design Issues
  • Storage requirement for recursive iteration in DSEA algorithm
    • Only six sequence terms need to be kept after each iteration.
  • Delta cannot be zero
    • k cannot be either p - 1 or p
    • Both a and b should be non-zero integers
  • Invertible Matrix M0
    • This can be guaranteed by choosing a and b correspondingly such that det(M0) ¹ 0.

Remarks
  • The GH-DH key agreement protocol and the signature generation in the GH-DSA scheme are successfully implemented in c++ and are tested thoroughly.
  • The implementation of signature verification is still in progress.
  • Several restrictions on system parameters and signing key selection are added to the system to increase system efficiency.
  • Implement GH-PKC on constrained devices, such as blackberry, to analyze performance.
